# 开源Android视频播放器开发方案

基于您的需求，我为您设计了一个完整的、可执行的开源Android视频播放器开发方案。这个方案旨在创建一个功能丰富、用户体验良好、易于维护的视频播放器应用。方案采用现代Android开发最佳实践，包括Kotlin语言、ExoPlayer播放库、Material Design UI等。整个项目将开源到GitHub，便于社区贡献。

## 1. 项目概述
- **项目名称**：AnVideoPlayer（或您自定义名称）
- **目标**：开发一个轻量级、功能强大的Android视频播放器，支持本地视频播放、网络视频流、播放列表管理、字幕支持、自定义主题等。
- **开源许可证**：Apache 2.0（简单、宽松，便于商业使用）。
- **目标用户**：Android用户，寻求替代系统播放器的开源解决方案。
- **预期功能**：
  - 视频播放（本地、网络）。
  - 播放控制（播放/暂停、快进/快退、音量控制）。
  - 播放列表管理。
  - 字幕支持（SRT格式）。
  - 自定义主题（深色/浅色模式）。
  - 后台播放。
  - 视频缩放（全屏、适应屏幕）。
  - 历史记录和收藏。

## 2. 技术栈
- **编程语言**：Kotlin（现代、简洁、安全）。
- **开发环境**：Android Studio（最新稳定版）。
- **播放库**：ExoPlayer + FFmpeg扩展（Google官方推荐，功能强大，支持多种格式；FFmpeg开源库提供底层编码解码支持，确保兼容性）。
- **UI框架**：Material Design 3（现代、美观）。
- **架构模式**：MVVM（Model-View-ViewModel，便于测试和维护）。
- **依赖注入**：Hilt（简化依赖管理）。
- **数据库**：Room（本地存储播放历史和收藏）。
- **异步处理**：Kotlin Coroutines + Flow。
- **网络请求**：Retrofit（如果需要网络视频API）。
- **其他库**：
  - Glide（图片加载，用于视频缩略图）。
  - RecyclerView（播放列表UI）。
  - MediaStore API（访问本地视频文件）。

## 3. 项目架构
- **模块化设计**：
  - `app`：主应用模块。
  - `core`：核心播放逻辑（ExoPlayer封装）。
  - `ui`：UI组件（播放器界面、列表）。
  - `data`：数据层（Room数据库、网络）。
- **文件夹结构**（基于Android项目标准）：
  ```
  app/
  ├── src/main/
  │   ├── java/com/example/anvideoplayer/
  │   │   ├── di/          # 依赖注入（Hilt模块）
  │   │   ├── data/        # 数据层（Repository、DAO）
  │   │   ├── domain/      # 业务逻辑（UseCase）
  │   │   ├── ui/          # UI层（Activity、Fragment、ViewModel）
  │   │   ├── core/        # 核心播放器（PlayerManager）

  ```
- **关键组件**：
  - `PlayerViewModel`：管理播放状态。
  - `VideoPlayerFragment`：播放器UI。
  - `PlaylistAdapter`：列表适配器。

## 7. 设置UI优化实现记录

### 7.1 需求分析
用户反馈：播放器原设置对话框所有功能混在一个页面中，容易导致内容看不见、变形等问题。

**解决方案**：
1. **分离设置页面**：将单一设置对话框分解为独立的Activity（每个功能分类一个），支持逐层递进式配置
2. **自动旋转优化**：根据视频的实际宽高比例自动选择横屏/竖屏播放
3. **旋转元数据处理**：读取ExoPlayer的视频旋转信息（unappliedRotationDegrees），正确应用宽高比判断

**2. PlayerActivity.kt**
- 新增：onVideoSizeChanged()监听器实现自动方向判断
  ```kotlin
  override fun onVideoSizeChanged(videoSize: VideoSize) {
    var w = videoSize.width
    var h = videoSize.height
    val rot = videoSize.unappliedRotationDegrees
    // 视频帧有90/270°旋转时，交换宽高用于比例计算
    if ((rot == 90 || rot == 270) && w > 0 && h > 0) {
      val tmp = w; w = h; h = tmp
    }
    if (w > 0 && h > 0 && playerPrefs.autoOrientationByVideo) {
      val ratio = w.toFloat() / h.toFloat()
      requestedOrientation = when {
        ratio >= 1.2f -> SCREEN_ORIENTATION_LANDSCAPE  // 16:9等宽屏
        ratio <= 0.8f -> SCREEN_ORIENTATION_PORTRAIT   // 9:16等竖屏
        else -> SCREEN_ORIENTATION_SENSOR              // 传感器决定
      }
    }
  }
  ```
- 比例阈值说明：
  - >= 1.2 → 强制横屏（适合16:9 1.78比例的视频）
  - <= 0.8 → 强制竖屏（适合9:16 0.56比例的视频）
  - 介于两者 → 传感器控制（设备倾斜决定）

**3. PlayerPrefs.kt**

  var autoOrientationByVideo: Boolean

### 7.3 关键技术点

**1. 自动方向判断算法**
```
计算视频宽高比 = width / height
- 若考虑旋转元数据：
  - 90°/270°旋转 → 交换宽高后再计算比例
  - 0°/180°旋转 → 直接使用

- 阈值逻辑：
  - 比例 >= 1.2 → 横屏（宽高大于1:1）
  - 比例 <= 0.8 → 竖屏（高宽大于1:1）
  - 中间值 → 传感器决定（用户可任意旋转设备）
```

**2. ActivityResult集成**
- PlayerSubtitleSettingsActivity使用registerForActivityResult(OpenDocument)选择字幕
- 选中后通过setResult(RESULT_OK, Intent.putExtra("subtitle_uri", ...))返回
- PlayerActivity的onActivityResult()可监听结果并加载字幕

**3. 设置数据流**
- 所有Activities直接操作PlayerPrefs（通过Hilt注入）
- 修改即时生效（写入SharedPreferences）
- PlayerActivity/PlayerSettingsDialog通过观察PlayerPrefs变化（可选：使用Flow）响应UI更新
